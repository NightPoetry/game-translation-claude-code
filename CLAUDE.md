# 游戏翻译项目指令

你正在帮助用户翻译一个游戏/应用。请严格遵循以下原则。

## 操作流程

详细步骤见 `translation-guide.md`，核心流程：

1. **先复制再翻译** — `cp -r 原版.app 中文版.app`，永远不动原版
2. **先配中文字体** — 复制系统中文字体到应用 fonts 目录，修改字体配置
3. **核心文件手动翻** — 启动脚本、UI界面等影响运行的文件
4. **场景文件用 Agent 并行翻** — 每批 4-6 个 Agent，每个负责一个文件
5. **质量检查** — 检查中文引号（会破坏语法）、检查翻译完整度

## 翻译原则

### 第零条：先判断场所

翻译前通过上下文判断当前场景的**场所/体裁**。场所决定语调：
- 街头骂战 → 粗口连发
- 论文/法庭 → 严谨书面
- 酒吧调情 → 暧昧挑逗
- 旁白叙述 → 中性客观

哪怕是小说里的一段论文，也要用论文语调翻译。

### 四条铁律

1. **骂人的词就翻成骂人的词** — 不要把 "fuck" 翻成 "该死"，"bitch" 翻成 "坏女人"。原文多脏翻译就多脏
2. **口语用短句** — 真人说话不用复杂从句
3. **情绪词优先于精确词** — 人在情绪中不挑最精确的词
4. **语调跟随场所切换** — 上下文变了笔调跟着变

### 禁止事项

- **禁止过度发挥** — 补充中文语境可以，添加原文没有的内容不行
- **禁止论文腔** — "达到了极限""在某种程度上"不是人话
- **禁止逻辑腔** — "如果A那么B，另一方面C"不是口语
- **禁止中性腔** — 不要所有角色都一个温度
- **禁止含蓄病** — 英文短词自带冲击力，中文需要补具体宾语才有同等力度
- **禁止使用中文引号 ""** — 会破坏 Ren'Py 等框架的字符串解析，只用英文单引号 ''

### 自检

翻完一句话问自己：
1. 像人话吗？
2. 温度匹配原文吗？
3. 有没有分析性词汇混进口语？
4. 有没有偏离原意？
