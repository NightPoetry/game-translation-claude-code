# 用 Claude Code 翻译游戏或应用

> 适用于：Ren'Py视觉小说、Electron应用、或任何包含文本文件的软件包

---

## 前置条件

- Claude Code CLI 已安装
- 目标应用已下载到本地
- 确认目标应用的文本文件格式（.rpy / .json / .xml / .txt 等）

---

## 第一步：创建副本（不干扰原版）

```bash
cp -r 原版.app 中文版.app
```

**为什么**：永远不动原文件。最终交付就是干干净净的两份——英文原版 + 中文版，互不干扰。翻译出问题可以随时用原版对比或回滚。

复制后立刻清理中文版的编译缓存（Ren'Py会把 .rpy 编译成 .rpyc 缓存，旧缓存可能覆盖你的翻译）：
```bash
find 中文版.app -name "*.rpyc" -delete
```

如果是 macOS .app 包，还要改 Info.plist 防止系统混淆：
```xml
<key>CFBundleName</key>
<string>XXX 中文版</string>
<key>CFBundleIdentifier</key>
<string>com.xxx.cn</string>
```

---

## 第二步：摸清文件结构

```bash
# 找出所有文本文件
find 中文版.app -name "*.rpy" -type f | wc -l

# 看几个文件了解格式
head -50 中文版.app/Contents/Resources/game/script.rpy
```

**要搞清楚的事**：
- 哪些文件包含用户可见文字？
- 哪些是纯代码不用翻译？
- 文件编码是什么？（UTF-8居多）
- 有没有已有的翻译框架（如Ren'Py的tl/目录）？

---

## 第三步：配置中文字体（关键！）

大多数英文应用自带的字体不支持中文，不配字体 → 全部显示方框。

### macOS 系统中文字体位置

```bash
# 查找可用的中文字体
ls /System/Library/Fonts/ | grep -i "heiti\|hiragino\|songti"
```

常用选择：
| 字体 | 路径 | 特点 |
|-----|------|------|
| 黑体 | `/System/Library/Fonts/STHeiti Medium.ttc` | 正文首选 |
| 冬青黑 | `/System/Library/Fonts/Hiragino Sans GB.ttc` | 美观 |
| 宋体 | `/System/Library/Fonts/Supplemental/Songti.ttc` | 传统印刷体 |

### 复制到应用目录并配置

```bash
cp "/System/Library/Fonts/Hiragino Sans GB.ttc" 中文版.app/Contents/Resources/game/fonts/
```

然后修改应用的字体配置文件。以Ren'Py为例，修改 `gui.rpy`：
```python
define gui.text_font = "fonts/Hiragino Sans GB.ttc"
define gui.name_text_font = "fonts/Hiragino Sans GB.ttc"
define gui.interface_text_font = "fonts/Hiragino Sans GB.ttc"
```

---

## 第四步：翻译文本文件

### 策略：手动翻核心 + Agent并行翻剩余

**核心文件**（直接影响游戏启动和UI）手动翻译，确保质量：
- 启动脚本（script.rpy）
- 角色选择（charSelection.rpy）
- 游戏循环（gameplay.rpy）
- UI界面（screens.rpy）—— 只改按钮文字，不动代码结构

**场景文件**用Claude Code的Agent工具并行翻译：
```
每批启动4-6个Agent，每个负责一个文件
```

### 给Agent的翻译指令模板

```
翻译 中文版.app/Contents/Resources/game/scene_xxx.rpy 文件，将英文翻译成中文。

翻译规则：
1. 保持代码结构不变（label, menu, show, $ 等命令）
2. 翻译所有对话和描述文字
3. 在保持原文意思不变的前提下，补充中文语境让表达更自然
4. 同时在 story_summaries 目录下创建同名双语故事摘要文件

不要修改变量名和文件路径引用。
```

---

## 第五步：质量检查

### 检查语法错误

```bash
# Ren'Py最常见的错误：中文引号破坏字符串
# 中文引号 "" 会被解析器误认为字符串边界
grep -rn '"' 中文版.app/Contents/Resources/game/*.rpy
```

**规则**：翻译文本中如需引号，只用英文单引号 `''`。

### 检查翻译完整度

```bash
# 找出未翻译的文件（与原版相同的文件）
for f in 原版.app/Contents/Resources/game/*.rpy; do
  fname=$(basename "$f")
  diff -q "$f" "中文版.app/Contents/Resources/game/$fname" >/dev/null 2>&1 && echo "未翻译: $fname"
done
```

### 运行测试

启动中文版应用，检查：
- [ ] 中文是否正常显示（不是方框）
- [ ] 菜单按钮是否正常
- [ ] 对话文本是否通顺
- [ ] 选项是否可以点击
- [ ] 游戏流程是否正常

---

## 第六步：常见问题速查

| 问题 | 原因 | 解决 |
|-----|------|------|
| 中文显示为方框 | 字体不支持CJK字符 | 第三步：复制系统中文字体 |
| `end of line expected` | 文本中含有中文引号"" | 替换为英文单引号'' |
| 界面菜单不显示 | screens.rpy代码结构被破坏 | 恢复原文，只翻译引号内的按钮文字 |
| .rpyc文件冲突 | 旧缓存与新翻译冲突 | 删除所有.rpyc文件，游戏启动时会重新编译 |
| 游戏启动崩溃 | 某个.rpy文件语法错误 | 看报错行号，检查该行引号/缩进 |

## 最终交付

```
项目目录/
├── 原版.app          ← 英文原版，从头到尾没动过
└── 中文版.app        ← 中文翻译版，独立运行
```

两份互不干扰，用户可以随时切换语言体验。

---

## 翻译原则速记

1. **先复制再翻译** — 永远不动原版
2. **先配字体再翻译** — 否则看不到效果
3. **核心手动、场景并行** — 效率和质量兼顾
4. **语气匹配原文** — 原文粗鲁就粗鲁，原文文雅就文雅
5. **补充语境但不编造** — 中文需要比英文更具体，但不加原文没有的意思
6. **翻完自问"像人话吗"** — 读一遍，不像口语就重写

## 说人话翻译指南

### 核心：你在翻译一个人说的话，不是在翻译一段文字

翻译台词时，脑子里要有一个活人站在你面前说这句话。他的表情、情绪、身份、粗鲁程度——都要听得到。

---

### 第零条：先看你在翻什么

翻译之前，通过上下文判断你正在翻的是什么**场所/体裁**。场所决定语调的基底。

| 场所 | 语调 | 特征 |
|-----|------|------|
| 街头对骂 | 粗口连发 | 脏字多、短句、不讲逻辑 |
| 酒吧调情 | 暧昧挑逗 | 双关语、暗示、拉扯感 |
| 法庭/论文 | 严谨书面 | 长句、专业术语、无情绪词 |
| 内心独白 | 意识流 | 碎片化、自问自答、犹豫 |
| 新闻报道 | 中性客观 | 无立场词、陈述事实 |
| 情书/告白 | 深情克制 | 短句但不粗、真诚、不油腻 |

**哪怕是小说里的一段论文，也要用论文的语调翻译**——因为在那个场景里，读者读到的就是一篇论文。语调跟随场所，不跟随作品整体风格。

同一部小说里可能同时出现街头骂战和学术演讲，翻译必须跟着切换。

---

### 四条铁律

**1. 骂人的词就翻成骂人的词**

AI翻译对脏话和骂人词汇天然保守，会自动降级：把"fuck"翻成"该死"，把"bitch"翻成"坏女人"。这是错的。原文骂人就是骂人，翻译必须保持同等力度。"fuck"就是"操"，"bitch"就是"婊子"。不要替原文文明化。

**2. 口语用短句，少用从句**

真人说话很少一口气说完一个复杂长句。断句要符合呼吸节奏。

**3. 情绪词优先于精确词**

人在情绪中不会挑最精确的词，而是挑最顺嘴的。

**4. 语调跟随场所切换**

翻译论文就用论文腔，翻译骂人就用骂人腔。不要用一个固定语调翻全文。上下文变了，你的笔也要跟着变。

### 反面案例（不要这样翻译）

| 原文 | ❌ 错误翻译 | 问题 | ✅ 正确翻译 |
|-----|-----------|------|-----------|
| "I can't respect him less" | "我对他已经看不起到了极限" | "极限"太书面，像写论文 | "我对那废物是零尊重" |
| "He will be so dumb if he believe it" | "如果他真信了，那就太蠢了" | "如果...那就..."太理智，像逻辑推理 | "他要真信了那就是蠢到家了" |
| "I hire you because you know how to convince people" | "我雇你就是因为你知道怎么说服人" | "说服"太正式，老板骂人时不会这么文绉绉 | "我雇你就是因为你会忽悠" |
| "Get down on your knees before I lose my patience!" | "在我失去耐心之前跪下来给我口交" | "给我口交"像医学描述，黑帮不这么说 | "在我失去耐心之前跪下来嗦我的大鸡巴" |
| "I want to see your naked bodies before you suck me" | "我想在看你们吸我之前看到你们裸露的身体" | "吸我"太含蓄断句不通顺，中文丢失了冲击力 | "把衣服脱了！我想在让你们嗦我的大鸡巴之前先欣赏你们的身体" |
| "enjoy the fuck of your life" | "享受你这辈子最爽的一次被猛干" | "被猛干"是对着中文二次发挥，偏离原意 | "享受你人生中最爽的一场性爱" |
| "I'm coming!!!" | "我来了！！！" | 中文高潮说"去"不说"来"，"来了"是到达的意思 | "我要去了！！！" |
| "a healthy heir for the company" | "为公司生个健康的继承人" | "继承人"是董事会用语，高潮时不会这么说 | "给公司生个胖小崽子" |
| "from the drama" | "从戏剧中休息一下" | "drama"在口语中是"破事/乱七八糟的事"，不是"戏剧" | "受够了那些破事" |
| "On the other hand" | "另一方面" | "另一方面"是论文连接词，口语不这么说 | "不过话说回来" |
| "What the fuck" | "搞什么" | 脏话严重降级 | "操你妈" / "搞什么鬼" |
| "Oh fuck no" | "该死，不会吧" | "Oh fuck"不是"该死" | "操，不会吧" |

### 中英文俚语差异速查

翻译时必须注意的中英文表达习惯差异，直译必错：

| 英文 | ❌ 直译 | ✅ 中文对应 | 原因 |
|-----|--------|-----------|------|
| I'm coming (性) | 我来了 | 我要去了 | 中文高潮用"去"，英文用"come" |
| heir | 继承人 | 崽子/种 | 脏话场景不用正式词 |
| drama | 戏剧 | 破事/狗血 | 口语中不是字面意思 |
| pet (dominatrix) | 宠物 | 贱货/小废物 | 女王语境下是贬义称呼 |
| stud spirit | 贴心的小宝贝 | 种马的气势 | stud = 种马，夸男人阳刚 |
| breeding (性癖) | 育种 | 受孕/配种 | "育种"太像畜牧术语 |
| donkey (指代) | 驴 | 驴（大屌暗语） | 保留隐喻即可 |
| donut (俚语) | 甜甜圈 | 菊花 | 中文用"菊花"指代肛门 |

### 诊断清单：翻完一句话后自检

- [ ] **像人话吗？** — 读出声，想象一个真人站在你面前说。别扭就重写
- [ ] **温度匹配吗？** — 原文是骂人还是调情还是叙述？你的中文是同一个温度吗？
- [ ] **有没有分析性词汇？** — "极限""程度""层面""方面"出现在口语里就是翻译腔
- [ ] **节奏对吗？** — 粗人说话是短句连发，不是一个长从句
- [ ] **有没有过度发挥？** — 原文没说的东西你加了吗？补充语境可以，编造不行
- [ ] **有没有太含蓄？** — 英文短词自带冲击力，中文直译会丢失。需要补具体宾语/程度词

---

### AI翻译的典型病

| 症状 | 表现 | 病因 | 治疗 |
|-----|------|------|------|
| **论文腔** | "达到了极限""在某种程度上" | AI习惯精确分析 | 换成口语：零、压根、根本 |
| **逻辑腔** | "如果A那么B，另一方面C" | AI习惯逻辑连接 | 断成短句，去掉连接词 |
| **中性腔** | 所有角色说话一个温度 | AI默认温和输出 | 先判断角色性格再选词 |
| **含蓄病** | "吸我""进入" | AI倾向委婉 | 直接用粗话：嗦、干、插、鸡巴 |
| **发挥病** | 原文没有的内容被添加 | AI试图"优化"原文 | 对照原文逐句检查，多的删掉 |

### 同一个词，不同角色说出来不一样

"fuck" 这个词：

| 角色 | 翻译 | 理由 |
|-----|------|------|
| 黑帮打手命令时 | "给老子干" | 粗暴支配 |
| 女主角求欢时 | "要我" / "进来" | 暧昧带情欲 |
| 旁白叙述时 | "发生了关系" | 中性客观 |
| 角色骂人时 | "操你妈" | 纯粹脏话 |

翻译前先问：**这句话是谁说的？他是什么人？他现在什么情绪？**
